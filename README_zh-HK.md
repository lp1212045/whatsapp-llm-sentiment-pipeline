# 🚀 自動化行銷數據流、LLM 情感分析與風險評級系統

## 📌 專案概述
本儲存庫包含一個全面的自動化數據處理系統，專為優化行銷營運、客戶情感分析及用戶風險評級而設計。透過整合 Python、大型語言模型（經由 Poe API 調用 Gemini）以及 Google Workspace API，此系統包含三大核心模組，能將每日非結構化的 WhatsApp 對話數據，自動轉化為具備商業價值的結構化洞察。

## 💼 商業價值
* **效率躍升：** 透過全自動化的數據清洗、API 請求與圖表更新，成功將每日營運數據的處理時間從 2-3 小時大幅縮減至僅 **15 分鐘**（節省超過 80% 的時間成本）。
* **主動式危機管理：** 運用 LLM 進行進階的情感分析，將客戶反饋進行精準分類（負面 / 正面 / 中立），即時標記潛在公關風險，以便客服團隊迅速介入處理。
* **自動化風險評級 (Risk Profiling)：** 追蹤用戶近 30 天的歷史行為軌跡，透過自訂演算法自動識別並標記高風險帳號（如：垃圾訊息、推銷帳號、競爭對手打手）。
* **動態儀表板更新：** 徹底消除人工輸入數據的需求，自動計算並聚合關鍵指標 (KPI)，並透過 API 批量推送至管理層的 Google Sheets 儀表板。

## 🛠️ 技術棧
* **開發語言：** Python 3.11 (`pandas`, `regex`, `datetime`)
* **AI 與自然語言處理：** Gemini-2.5-Flash (經由 Poe API), 提示詞工程 (Prompt Engineering)
* **雲端與系統串接：** Google Drive API (檔案自動抓取), Google Sheets API / `gspread` (讀取動態配置與數據匯出)
* **效能與穩定性優化：** `concurrent.futures` (多執行緒加速), 自訂 Exponential Backoff 裝飾器 (處理 API 請求限制與重試機制)
* **自動化部署：** GitHub Actions (CRON 定時任務、pip 依賴快取加速、`workflow_dispatch` 參數化手動觸發)

## ⚙️ 核心架構與模組

### 1. `main.py` (核心數據管道與 LLM 引擎)
* **數據獲取：** 透過模組化的 Google 服務帳戶安全驗證，動態搜尋並從 Google Drive 下載每日原始對話紀錄。
* **過濾與去重：** 根據動態讀取的關鍵字配置，結合時間容錯率與用戶身分權重，執行嚴格的文字去重與特徵比對。
* **LLM 處理 (多執行緒)：** 將過濾後的對話發送至 LLM，過濾垃圾意圖並執行精細的品牌情感分析，最後將純淨數據寫入每日報表。

### 2. `risk_analysis.py` (用戶行為與風險分析引擎)
* **行為聚合：** 回溯並解析過去 30 天的數據，建立活躍用戶的行為輪廓（如：加入群組數量、特定品牌提及頻率）。
* **規則評分機制：** 套用自訂的風險評分演算法計算用戶危險指數。
* **標籤化管理：** 自動更新 "Sim Master" 資料庫，將用戶自動分級為 `Real` (真實)、`Watch` (觀察)、`Business` (推銷) 或 `IFT Seeder` (打手)，大幅降低社群維護成本。

### 3. `dashboard.py` (儀表板自動化模組)
* **數據聚合：** 統整每日各群組的活躍度，將討論主題進行分類，並計算各大品牌的正負面聲量佔比。
* **批量更新 (Batch Update)：** 調用 `batchUpdate` API 高效精準地覆寫 Google Sheets 上的特定儲存格，實現視覺化圖表的無人化更新。

## 🔒 資訊安全與系統穩定性聲明
* **資料隱私：** 基於企業合規性考量，所有敏感配置（包含 Google 憑證、API 金鑰、資料夾 ID 等）均嚴格透過環境變數 (Environment Variables) 與 GitHub Secrets 進行隱密管理。本公開儲存庫絕對不包含任何真實的客戶隱私數據。
* **容錯機制：** 所有 Google API 呼叫皆由自訂的 `@with_retry` 裝飾器包覆，實作了 **指數退避 (Exponential Backoff)** 演算法，確保系統在遭遇 API 配額限制或 `429 Too Many Requests` 時，仍能穩定運行不崩潰。
